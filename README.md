# CS553_2026 - Distributed Algorithms Simulator

This repository is a course project for CS553 at UIC. It consumes graphs generated by NetGameSim and turns them into a running actor-based message passing simulation.

## Overview

The simulator has one main path:

```text
NetGameSim graph export -> graph enrichment -> Akka actor runtime -> Echo/Wave results
```

NetGameSim is used for graph generation only. The Scala code in this repository loads the exported graph, adds message labels and node traffic distributions, creates the actor system, and records run metrics.

## Features

- **NetGameSim input**: Load exported NetGameSim graphs into the internal graph model.
- **Actor mapping**: Create one Akka classic actor per graph node and one outgoing channel per graph edge.
- **Edge labels**: Drop and count messages that are not allowed on the selected edge.
- **Node PDFs**: Configure per-node message production probabilities.
- **Initiators**: Start computation from timers or injected input.
- **Assigned algorithms**:
  - **Echo**: Broadcast and convergcast on a topology with reverse CONTROL paths.
  - **Wave**: Flooding-style wave that records one observation per reached node.
- **Outputs**: Log run activity, print metrics summaries, and export enriched graph artifacts.

## Getting Started

### Building the Project

```bash
git submodule update --init --recursive
sbt compile
sbt test
```

### Running the Graph-Driven Simulator

NetGameSim is included as a submodule under `netgamesim/`. Use it only for graph generation and export.

The checked-in NetGameSim config is set to produce JSON-compatible graph artifacts in `netgamesim/outputs/`.

```bash
cd netgamesim
sbt "runMain com.lsc.Main realgraph"
cd ..
```

For directed NetGameSim graphs, use the wave-only config:

```bash
sbt "runMain com.uic.cs553.distributed.cli.SimMain --graph netgamesim/outputs/realgraph.ngs --config conf/wave-directed.conf --out outputs/wave-run"
```

Use the Echo+Wave config only with a topology that supports reverse CONTROL paths:

```bash
sbt "runMain com.uic.cs553.distributed.cli.SimMain --graph examples/bidirectional-echo-graph.json --config conf/echo-wave-bidirectional.conf --out outputs/echo-wave-run"
```

The `--out` directory receives serialized enriched artifacts:

```text
enriched-graph.json   # topology plus edge labels, node PDFs, initiators, algorithms
enriched-graph.dot    # visualization with edge-label message types
```

Interactive input mode accepts commands from standard input:

```bash
sbt "runMain com.uic.cs553.distributed.cli.SimMain --graph examples/bidirectional-echo-graph.json --config conf/interactive-echo-wave.conf --interactive"
```

Example interactive commands:

```text
inject 0 WORK manual-job
inject 1 PING hello
quit
```

Optional injection file:

```bash
sbt "runMain com.uic.cs553.distributed.cli.SimMain --graph netgamesim/outputs/realgraph.ngs --config conf/wave-directed.conf --inject /path/to/injections.txt"
```

Injection file format:

```text
# delayMs nodeId kind payload
0 0 WORK bootstrap-job
500 2 PING hello
```

## Runtime Algorithm Plug-Ins

The graph-driven runtime uses `DistributedAlgorithm` implementations under `runtime/`. A node algorithm receives a `NodeContext` with the local node id, outgoing neighbors, send helpers, logging, and algorithm event emission.

To create a new runtime algorithm:

```scala
import com.uic.cs553.distributed.runtime.*

final class MyRuntimeAlgorithm(nodeId: Int, initiatorNode: Int) extends DistributedAlgorithm {
  override val name: String = "my-algorithm"

  override def onStart(ctx: NodeContext): Unit =
    if (nodeId == initiatorNode) {
      ctx.sendToAll("CONTROL", "my-algorithm:start")
      ctx.emitAlgorithmEvent(name, "started")
    }
}
```

## Key Concepts

### Graph-Driven Runtime

`SimulationBootstrap` creates one Akka classic actor per graph node. Each node receives only its outgoing graph neighbors as `ActorRef` channels.

### Edge Labels

Allowed message kinds are configured per edge. Node sends are dropped and counted if the chosen message kind is not permitted on the selected edge.

### PDFs

Each node has a configurable probability distribution over message kinds. Timer nodes sample from this distribution unless configured with a fixed message kind.

### Initiators

Timer initiators periodically emit messages. Input initiators accept file-driven or interactive external injections.

### Cinnamon / Akka Insights

The application contains Cinnamon actor instrumentation configuration, but the sbt plugin is disabled by default because Cinnamon artifacts require Akka commercial credentials and a tokenized resolver. See `docs/CINNAMON.md` for the exact enablement steps.
